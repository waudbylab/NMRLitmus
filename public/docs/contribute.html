Contributing

Adding new buffer data to the database

Contributions of new buffer chemical shift data are very welcome. The database is built from YAML source files in the data/ directory of the GitHub repository. Please open a pull request with your additions.

Attribution: Contributions are individually attributed. Each dataset carries author names, ORCIDs, and affiliations, which appear on the Reference data page and in any citations generated by the app.

Repository layout

Data are organised into named subdirectories, one per measurement campaign (e.g. data/valarie_d2o/). Each subdirectory contains:

  • sample.yaml — experimental metadata for the whole dataset
  • one .yaml file per buffer (e.g. acetate.yaml, imidazole.yaml)

sample.yaml

Describes the experimental conditions, calibration methods, and authors for a set of measurements. A minimal example (from data/valarie_d2o/sample.yaml): 

solvent: 100% D2O
authors:
  - name: Chris Waudby
    email: c.waudby@ucl.ac.uk
    orcid: 0000-0001-7810-3753
    affiliation: UCL School of Pharmacy
  - name: Valarie Mitala
    email: valarie.mitala.22@ucl.ac.uk
    affiliation: UCL School of Pharmacy
date_measured: '2025-12-28'
reference_temperature_K: 298
temperature_calibration: d4-methanol chemical shift thermometer
pH_calibration: >-
  Hanna HI-2002 Edge pH Meter, calibrated against NIST-traceable buffers pH
  4.01, 7.00, 10.01, thermostatted at 298 K. Reading pH*.
ionic_strength_control: 'Addition of 0, 150 or 300 mM NaCl'

Buffer YAML files

Each buffer file specifies the molecule and provides a tab-delimited data block of observed chemical shifts measured across a range of pH, temperature, and ionic strength conditions. The column header encodes the nucleus, resonance label, and multiplicity (e.g. 1H(Ha,s) for a singlet 1H resonance labelled Ha).

Example (from data/valarie_d2o/imidazole.yaml, data truncated):

buffer_name: Imidazole
smiles: C1=CN=CN1
ionisation_states: 2
protonated_charge: 1
data: |

The columns are: ionic strength I (M), temperature T (K), pH, then one column per resonance. Missing values can be indicated with missing.

How to contribute

  1. Fork the repository on GitHub.
  2. Create a new subdirectory under data/ for your dataset.
  3. Add a sample.yaml with your author information and experimental details.
  4. Add one YAML file per buffer with your chemical shift data.
  5. Open a pull request — we will review, run the fitting pipeline, and merge.

If you are unsure about the format, feel free to open an issue on GitHub and we will help you prepare your submission.
